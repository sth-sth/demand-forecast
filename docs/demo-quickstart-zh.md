# Demo 快速上手（中文）

## 1. 目标

本 Demo 用于让团队在不准备任何外部数据的情况下，快速完成一次需求预测全流程：

1. 加载示例数据
2. 查看示例数据内容
3. 理解字段中文含义
4. 一键运行预测任务
5. 查看模型对比结果与可视化

## 2. 一键体验步骤

1. 启动平台：
   - `cp .env.example .env`
   - `docker compose up --build`
2. 打开前端：`http://localhost:8080`
3. 点击 `加载 Demo 数据`：
   - 系统会自动注册内置数据集
   - 自动展示数据预览（前 15 行）
4. 点击 `一键跑 Demo`：
   - 自动创建预测任务
   - 自动轮询任务状态
   - 完成后展示排行榜和冠军模型曲线

## 3. Demo 数据说明

Demo 文件：`backend/data/demo/demand_demo_cn.csv`

字段说明：

- `日期`
  - 含义：业务日期
  - 对应参数：`time_col`
- `销量`
  - 含义：实际需求量
  - 对应参数：`target_col`
- `商品编码`
  - 含义：时序ID（不同商品/门店）
  - 对应参数：`item_col`
- `是否促销`
  - 含义：促销标记，1 表示促销，0 表示非促销
- `是否缺货`
  - 含义：缺货标记，1 表示当日供给异常
- `价格`
  - 含义：销售价格（业务解释字段）
- `温度`
  - 含义：环境温度（业务解释字段）
- `节假日标签`
  - 含义：是否节假日（业务解释字段）

## 4. 输入参数中文解释

- `数据集（dataset_id）`
  - 选择本次建模用哪个数据集。
- `预测步长（horizon）`
  - 往后预测多少期；日频下常用 7、14、30。
- `优化指标（metric）`
  - 模型排序标准，冠军模型按该指标最小值选出。
- `调参轮次（tune_trials）`
  - 自动调参搜索次数；数值越大，效果可能更好但耗时更高。
- `自动对比全部模型（use_all_models）`
  - 打开后系统会遍历全部可用模型。

## 5. Demo 相关接口

- `POST /api/datasets/demo`
  - 加载内置 Demo 数据集。
- `GET /api/datasets/{dataset_id}/preview?limit=15`
  - 获取数据预览内容。
- `POST /api/runs`
  - 创建并运行预测任务。
- `GET /api/runs/{run_id}`
  - 查询任务状态。
- `GET /api/runs/{run_id}/results`
  - 查询模型明细结果。
- `GET /api/runs/{run_id}/visualizations`
  - 获取可视化图表 JSON。

## 6. 使用建议

- 新人培训：先跑 Demo，再接入真实业务数据。
- 业务评审：以 Demo 字段解释为模板，补齐你们的数据字典。
- 上线前：先用 `use_all_models=true` 跑一次全量比较，再固化候选模型名单。
